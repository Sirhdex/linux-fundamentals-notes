## Concepts
- Allows users and admins to automate tasks by running them at specific times, elimnating the need for manual intervention
- Can serve as both valued admin tool and a valid attack vector (undocumented cronjobs / maintaining backdoors)
- Timers can be constructed through vim and sent into directories by use for management
- We then configure our Timer through a script
- systemctl can then be used to enable and start the newly created process
- Main differnece between Cron and Systemd = how they're configured
